## 1. Charger le dataset fil rouge

```{webr}
#| autorun: true
# sf = format spatial vectoriel (polygones de regions) ; dplyr = manipulation tableaux ;
# tibble = tableau moderne. Ces 3 packages forment la base J10.
library(sf); library(dplyr); library(tibble)

# Reproductibilite computationnelle : meme chemin -> meme resultat, sur toute machine.
# Ici WebR a pre-charge le .gpkg dans la VM du navigateur (chemin relatif fonctionne).
regions <- sf::read_sf("_commons/data/jour_10_extraits/regions_indicateurs_demo.gpkg")

# Diagnostic minimal : nombre d'objets, systeme de coordonnees, colonnes disponibles.
cat("Nombre de regions :", nrow(regions), "\n")
cat("CRS               :", sf::st_crs(regions)$input, "\n")
cat("Colonnes          :", paste(names(regions), collapse = ", "), "\n")
```

## 2. Inspecter le tableau d'indicateurs

```{webr}
# st_drop_geometry() = on retire la colonne geometrie pour ne voir que les attributs.
# select() = on garde 7 colonnes utiles ; arrange(desc(...)) = tri par population decroissante.
regions |>
  sf::st_drop_geometry() |>
  dplyr::select(region, population, menages, eau_amelioree_pct,
                alphabetisation_pct, ecoles, formations_sanitaires) |>
  dplyr::arrange(dplyr::desc(population))
```

## 3. Carte statique de l'accès à l'eau améliorée

Carte choroplethe : la couleur de chaque region encode le pourcentage de menages ayant acces a une source d'eau amelioree (forage, robinet, borne-fontaine).

```{webr}
# ggplot2 = grammaire des graphiques ; scales = utilitaires (transparence, format).
# On utilise ggplot2 ici (et non tmap) car tmap n'est pas portable WebR.
library(ggplot2)
library(scales)

# geom_sf trace les polygones ; aes(fill=...) lie la variable a la couleur ;
# geom_sf_label pose le nom de chaque region. Palette viridis = lisible daltoniens.
ggplot(regions) +
  geom_sf(aes(fill = eau_amelioree_pct), colour = "white", linewidth = 0.3) +
  geom_sf_label(aes(label = region), size = 2.2, colour = "black",
                fill = scales::alpha("white", 0.7),
                label.padding = unit(0.1, "lines")) +
  scale_fill_viridis_c(name = "Eau amelioree (% menages)", option = "viridis") +
  labs(title = "Acces a l'eau amelioree par region - Cameroun",
       subtitle = "Source dataset Edith Darin - indicateurs d'entrainement (10 regions ADM1)",
       caption = "Source : regions_indicateurs_demo (Edith Darin, GDSG) - IFORD x GDSG 2026 - Donnees d'entrainement, non officielles") +
  theme_minimal() +
  theme(axis.text = element_blank(), axis.ticks = element_blank(),
        panel.grid = element_blank())
```

## 5. Automatisation par boucle régionale

Fonction unitaire qui calcule densité + taille de ménage + FOSA/10k pour une région donnée.

```{webr}
# purrr = programmation fonctionnelle (alternative aux boucles for).
library(purrr)

# Fonction "unitaire" : un nom de region -> un tibble d'une ligne.
# Pattern reutilisable : on encapsule le calcul, on l'appellera N fois ensuite.
resumer_region <- function(nom) {
  r <- regions |> dplyr::filter(region == nom)
  if (nrow(r) == 0) {
    return(tibble::tibble(region = nom, densite = NA_real_,
                          taille_menage = NA_real_,
                          fosa_pour_10k = NA_real_))
  }
  # st_area renvoie des m^2 ; /1e6 convertit en km^2.
  surface_km2 <- as.numeric(sf::st_area(r)) / 1e6
  tibble::tibble(
    region        = nom,
    pop_2024      = r$population,
    surface_km2   = round(surface_km2, 0),
    densite       = round(r$population / surface_km2, 1),    # hab/km^2
    taille_menage = round(r$population / r$menages, 2),      # personnes/menage
    fosa_pour_10k = round(r$formations_sanitaires / r$population * 1e4, 2)  # FOSA pour 10k hab
  )
}

# Test unitaire avant de boucler : on verifie sur une seule region.
resumer_region("Adamaoua")
```

## 6. Appliquer aux 10 régions en une commande

```{webr}
# map_dfr = "map data-frame row-bind" : applique la fonction a chaque element du
# vecteur et empile les resultats en un seul tibble. Equivalent fonctionnel d'un
# for + bind_rows, mais plus court, plus lisible, et type (verifie les colonnes).
tableau_regions <- purrr::map_dfr(regions$region, resumer_region) |>
  dplyr::arrange(dplyr::desc(densite))

tableau_regions
```

## 8. Les 7 métadonnées minimales d'un livrable

Voici la grille à coller dans chaque export FAIR (Section 6 de `demo.qmd`).

```{webr}
# Principes FAIR (2016) : Findable - Accessible - Interoperable - Reusable.
# Les 7 elements ci-dessous sont la traduction concrete minimale pour un livrable
# geospatial : sans eux, ton .gpkg/.csv n'est pas reutilisable (R du FAIR).
metadonnees_livrable <- tibble::tribble(
  ~`#`, ~`Element`,           ~`Valeur exemple`,
  1L,   "Titre",              "Mon indicateur compose par region",
  2L,   "Source",             "regions_indicateurs_demo.gpkg (Edith Darin, GDSG)",
  3L,   "Date production",    format(Sys.Date(), "%Y-%m-%d"),
  4L,   "CRS",                "EPSG:4326 (WGS84)",
  5L,   "Echelle/resolution", "ADM1 - 10 regions du Cameroun",
  6L,   "Methode",            "decrire la formule de l'indicateur",
  7L,   "Limites",            "donnees d'entrainement, ne pas citer comme officiel"
)
metadonnees_livrable
```

::: {.iford-callout}
**Règle simple** : sans ces 7 informations, ton `.gpkg` ou `.csv` est inutilisable par quelqu'un d'autre dans 6 mois.
:::
