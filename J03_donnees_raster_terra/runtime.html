J3 — Données raster avec terra

Code R exécutable dans le navigateur — vecteur + raster lecture seule

Mode d’emploi. Les cellules Run code (cellules bleues {webr}) tournent dans le navigateur — clique pour exécuter. Les blocs r simples (non bleus) sont en lecture seule : ils montrent le code raster à étudier et à exécuter dans RStudio desktop.

Pourquoi cette séparation ? Le package R terra (raster + GDAL/PROJ/GEOS compilés en WASM, ~15 Mo) déclenche une régression connue de WebR 0.6 : TypeError: resolved is not a function au moment du téléchargement. Tant que ce bug amont n’est pas corrigé, la partie raster ne peut pas s’exécuter dans le navigateur — mais les lignes restent visibles pour la lecture et la copie vers desktop.

Données 100 % réelles : limites administratives officielles GADM v4.1, MNT SRTM agrégé 30 arc-sec (~1 km) du Cameroun (NASA/USGS).

3. Vecteur vs raster

Le vecteur (points, lignes, polygones) décrit des entités discrètes : un département, une route, une ville. Le raster (grille de pixels) décrit des phénomènes continus : altitude, température, densité, indice de végétation.

Pour la plupart des analyses spatiales utiles à la statistique publique, on combine les deux : on prend un raster continu (par exemple un MNT SRTM ou une grille WorldPop), et on l’agrège par polygone administratif (extraction zonale) pour obtenir une valeur par région ou par département.

Les sections 3 bis à 3 octies ci-dessous présentent le pipeline complet terra à reproduire dans RStudio desktop. Le code y est commenté ligne à ligne. Note : sur un poste avec R installé, ouvrir demo.qmd du même dossier exécute ce pipeline en direct.

3 bis. Charger terra et le SRTM Cameroun

# sf = vecteur ; terra = raster (grille de pixels, successeur de l'ancien package raster).
library(sf)
library(terra)
library(dplyr)
library(ggplot2)

# Limites administratives du Cameroun (GADM v4.1).
adm0 <- read_sf("../_commons/data/gadm41_CMR_0.json")

# MNT = Modele Numerique de Terrain : une image ou chaque pixel = une altitude.
# SRTM = mission satellite NASA/USGS qui a mesure l'altitude du globe en 2000.
# rast() ouvre le fichier sans tout charger en RAM (juste un pointeur sur disque).
srtm <- rast("../_commons/data/cmr_srtm/srtm_cmr_30s.tif")
names(srtm) <- "elevation_m"
srtm

3 ter. Inspecter le SpatRaster

# Resolution = taille d'un pixel au sol. nlyr = nombre de "couches" (bandes).
# Pour un MNT, 1 bande suffit (l'altitude) ; une image satellite peut en avoir 13.
c(resolution = res(srtm)[1],
  nb_cellules = ncell(srtm),
  nb_bandes   = nlyr(srtm),
  EPSG        = crs(srtm, describe = TRUE)$code)

# ext() = emprise geographique (xmin, xmax, ymin, ymax) couverte par le raster.
ext(srtm)

3 quater. Crop + mask sur la silhouette du Cameroun

# crop() = decoupe le raster a l'emprise rectangulaire (bbox) du vecteur.
# mask() = met NA sur tous les pixels hors des polygones (silhouette pays).
# On enchaine TOUJOURS crop puis mask : crop reduit la taille, mask nettoie les bords.
srtm_cmr        <- crop(srtm, adm0)
srtm_cmr_masked <- mask(srtm_cmr, vect(adm0))

plot(srtm_cmr_masked,
     main = "Altitude (m) — Cameroun",
     plg  = list(title = "m"),
     sub  = "Source : NASA SRTM 30 arc-sec · IFORD x GDSG 2026")

3 quinquies. Algèbre raster — zones de basse altitude

# Algebre raster : on applique une operation a chaque pixel (cellule par cellule).
# Ici "< 500" renvoie 1 (vrai) si l'altitude est < 500 m, 0 sinon, NA hors masque.
basse_alt <- srtm_cmr_masked < 500
plot(basse_alt,
     main = "Zones de basse altitude (< 500 m) — Cameroun",
     plg  = list(title = "classe"),
     sub  = "Source : NASA SRTM 30 arc-sec · IFORD x GDSG 2026")

3 sexies. Classification multi-niveaux

# Classification = regrouper les pixels par tranches (ici 4 classes d'altitude).
# ifel() est l'equivalent raster de if_else() : SI condition ALORS x SINON y.
zones <- ifel(srtm_cmr_masked < 200, 1,
        ifel(srtm_cmr_masked < 500, 2,
        ifel(srtm_cmr_masked < 1000, 3, 4)))
plot(zones,
     main = "Classes d'altitude — Cameroun",
     plg  = list(title = "classe (1=<200m, 2=200-500, 3=500-1000, 4=>1000)"),
     sub  = "Source : NASA SRTM 30 arc-sec · IFORD x GDSG 2026")

3 septies. Statistique globale

# global() = statistique calculee sur TOUS les pixels valides du raster.
# Renvoie un data.frame ; ici on arrondit en m.
global(srtm_cmr_masked, c("mean", "min", "max", "sd"), na.rm = TRUE) |> round()

3 octies. Extraction zonale par région + choroplèthe

adm1 <- read_sf("../_commons/data/gadm41_CMR_1.json")

# Extraction zonale = pour chaque polygone (region), on resume les pixels
# qui tombent dedans (ici la moyenne d'altitude). Pont raster <-> vecteur.
elev_par_region <- terra::extract(srtm_cmr_masked, vect(adm1),
                                   fun = "mean", na.rm = TRUE)
elev_par_region$NAME_1 <- adm1$NAME_1

elev_par_region |>
  select(NAME_1, elev_moy = elevation_m) |>
  mutate(elev_moy = round(elev_moy)) |>
  arrange(desc(elev_moy))

# Choroplethe = carte ou la couleur de chaque polygone code une valeur numerique.
adm1_elev <- adm1 |>
  mutate(elev_moy = round(elev_par_region$elevation_m))

ggplot(adm1_elev) +
  geom_sf(aes(fill = elev_moy), color = "white", linewidth = 0.3) +
  scale_fill_viridis_c(option = "G", name = "Élévation\nmoyenne (m)") +
  geom_sf_label(aes(label = NAME_1), size = 2.1,
                color = "white", fontface = "bold") +
  labs(title    = "Élévation moyenne par région — Cameroun",
       subtitle = "MNT SRTM 30 arc-sec (NASA/USGS) · terra::extract",
       caption  = "Source : NASA SRTM 30 arc-sec · GADM v4.1 · IFORD x GDSG 2026") +
  theme_minimal() +
  theme(axis.text = element_blank(), axis.ticks = element_blank(),
        panel.grid = element_blank())

## 4. Substitut vecteur — superficie des régions

En attendant que terra soit utilisable en WebR, voici l'équivalent vecteur du pattern d'extraction zonale : on calcule une grandeur **par polygone** (ici la superficie) directement avec `sf`.

```{webr}
# Calcul de la superficie de chaque region en km^2 (operation 100% sf).
# On reprojete d'abord en UTM 32N (metres) pour mesurer correctement.
adm1_utm <- st_transform(adm1, 32632)
adm1_utm$superficie_km2 <- as.numeric(st_area(adm1_utm)) / 1e6

# Classement des regions par superficie decroissante.
adm1_utm |>
  st_drop_geometry() |>
  select(NAME_1, superficie_km2) |>
  mutate(superficie_km2 = round(superficie_km2)) |>
  arrange(desc(superficie_km2))
```

## 5. Carte choroplèthe — superficie des régions
