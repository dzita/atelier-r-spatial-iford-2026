## 1. R comme calculatrice

```{webr}
#| autorun: true
# Un *vecteur* est une séquence ordonnée de valeurs de même type.
# On le crée avec c() (= "combine"). Ici un vecteur de chaînes de caractères.
regions <- c("Adamaoua", "Centre", "Est", "Extrême-Nord", "Littoral",
             "Nord", "Nord-Ouest", "Ouest", "Sud", "Sud-Ouest")
# Vecteur numérique parallèle : 1 valeur par région, dans le MÊME ordre.
pop_est <- c(1340000, 4400000, 870000, 4200000, 3400000,
             2700000, 2000000, 2000000, 800000, 1700000)

length(regions)   # nombre d'éléments
sum(pop_est)      # somme totale (population du Cameroun)
mean(pop_est)     # moyenne arithmétique par région
```

**À essayer :** changez les valeurs de `pop_est`, ou ajoutez une 11ᵉ région imaginaire — puis relancez.

## 2. Vecteurs et opérations vectorielles

```{webr}
# R vectorise : on divise les 10 valeurs d'un coup, pas besoin de boucle.
pop_millions <- pop_est / 1e6
round(pop_millions, 2)   # arrondi à 2 décimales

# Indexation booléenne : on garde les noms là où la condition est vraie.
regions[pop_est > 3e6]
```

## 3. Tibble : la table de données moderne

```{webr}
library(tibble)   # tibble = tableau moderne, affichage plus lisible
library(dplyr)    # verbes de manipulation : filter, mutate, select...

# On crée un tibble : un tableau où chaque colonne est un vecteur de même longueur.
tbl_cmr <- tibble(
  region   = regions,
  pop_2019 = pop_est,
  capitale = c("Ngaoundéré", "Yaoundé", "Bertoua", "Maroua", "Douala",
               "Garoua", "Bamenda", "Bafoussam", "Ebolowa", "Buea")
)

tbl_cmr   # afficher le tableau
```

## 4. Les verbes dplyr en pratique démographique

```{webr}
# Le pipe |> se lit "puis". On enchaîne les verbes dplyr de haut en bas.
tbl_cmr |>
  mutate(                                # mutate = ajouter / modifier une colonne
    pop_millions = pop_2019 / 1e6,
    classe = case_when(                  # case_when = IF / ELSE IF / ELSE lisible
      pop_2019 < 1e6 ~ "Moins de 1M",
      pop_2019 < 3e6 ~ "1 à 3M",
      TRUE           ~ "Plus de 3M"      # TRUE = clause par défaut
    )
  ) |>
  arrange(desc(pop_2019))                # tri décroissant par population
```

```{webr}
# filter = garder certaines LIGNES (selon une condition).
# select = garder certaines COLONNES (par leur nom).
tbl_cmr |>
  filter(pop_2019 > 3e6) |>
  select(region, capitale, pop_2019)
```

## 5. Indicateurs DHS Cameroun 2018 (vraies données)

On charge directement le CSV agrégé par région issu de l'API DHS StatCompiler (`rdhs::dhs_data`). Le fichier a été pré-chargé dans la VM WebR au démarrage de la page (déclaré dans `webr.resources` du YAML) — pas de téléchargement runtime, ça marche en ligne **et** hors-ligne en preview locale.

```{webr}
library(tidyr)   # pivot_wider / pivot_longer pour réorganiser un tableau

# Lecture du CSV pré-chargé dans le runtime WebR (pas de réseau ici).
dhs <- read.csv("_commons/data/dhs_cmr/indicateurs_dhs_cmr_2018.csv")

# Le CSV est en format LONG (1 ligne = 1 indicateur d'1 région).
# pivot_wider transforme en format LARGE (1 ligne = 1 région, 1 col par indicateur).
dhs_wide <- dhs |>
  select(region, indicateur, valeur) |>
  pivot_wider(names_from = indicateur, values_from = valeur) |>
  rename(tx_eau_ameliore = WS_SRCE_H_IMP,   # on renomme les codes DHS
         tx_electricite  = HC_ELEC_H_ELC,   # en noms parlants en français
         tx_alpha_femmes = ED_LITR_W_LIT,
         tx_alpha_hommes = ED_LITR_M_LIT,
         taille_menage   = HC_HHSZ_H_AVG)
dhs_wide
```

```{webr}
# On crée un indicateur dérivé (écart H-F) puis on classe par électrification.
dhs_wide |>
  mutate(ecart_alpha_hf = round(tx_alpha_hommes - tx_alpha_femmes, 1)) |>
  arrange(desc(tx_electricite)) |>
  select(region, tx_alpha_femmes, tx_electricite,
         taille_menage, ecart_alpha_hf)
```

**À retenir.** `filter` filtre les lignes, `select` choisit les colonnes, `mutate` crée une variable, `group_by + summarise` agrège, `pivot_wider` réorganise. Cinq verbes pour 80 % de l'analyse de données. Les valeurs ci-dessus sont **les vraies estimations DHS 2018** déjà pondérées par le plan de sondage.

## 6. Première carte — vraies limites GADM Cameroun

Le fichier GADM v4.1 du Cameroun (10 régions, ~190 Ko) est pré-chargé dans la VM WebR au démarrage de la page. On l'ouvre directement avec `sf::read_sf()` — pas de réseau au runtime, ça marche aussi quand tu n'as pas internet.

```{webr}
#| autorun: true
library(sf)        # sf = "simple features" : objets spatiaux vectoriels
library(ggplot2)   # grammaire graphique de référence en R
library(dplyr)

# read_sf() lit un fichier SIG (ici GeoJSON) et renvoie un objet sf =
# un data.frame + une colonne "geometry" attachée à chaque ligne.
adm1 <- read_sf("_commons/data/gadm41_CMR_1.json")

nrow(adm1)              # nombre d'entités spatiales (10 régions attendues)
st_crs(adm1)$epsg       # code EPSG du système de coordonnées (4326 = WGS 84)
head(adm1$NAME_1)       # quelques noms de régions
```

Et maintenant la vraie carte des 10 régions du Cameroun, avec les contours administratifs officiels :

```{webr}
ggplot(adm1) +
  # geom_sf dessine la géométrie ; fill = NAME_1 attribue 1 couleur par région.
  geom_sf(aes(fill = NAME_1), color = "white", linewidth = 0.3, show.legend = FALSE) +
  # geom_sf_label pose une étiquette au centroïde de chaque polygone.
  geom_sf_label(aes(label = NAME_1),
                size = 2.4, colour = "black",
                fill = scales::alpha("white", 0.75), label.size = 0,
                fontface = "bold", label.padding = unit(0.1, "lines")) +
  # Palette viridis discrète : daltonien-friendly, lisible en N&B.
  scale_fill_viridis_d(option = "G") +
  labs(title = "Les 10 régions du Cameroun",
       subtitle = "Limites administratives ADM1 — GADM v4.1",
       caption = "Source : GADM v4.1 · Atelier IFORD × GDSG 2026") +
  theme_minimal()   # retire les axes/grille parasites
```

## 6 bis. Choroplèthe densité — carte thématique complète

On combine la géométrie ADM1 avec le tibble de population créé en section 3, on calcule la densité (habitants par km²), et on colore.

```{webr}
# Une projection (CRS) "plane" en mètres est NÉCESSAIRE pour mesurer surfaces
# et distances. EPSG:32632 = UTM zone 32N, adapté à l'ouest du Cameroun.
adm1_utm <- st_transform(adm1, 32632)

# st_area renvoie une surface en m² (unité du CRS) ; on convertit en km².
adm1_utm$superficie_km2 <- as.numeric(st_area(adm1_utm)) / 1e6

# Contrôle de cohérence : total attendu ~475 000 km² (superficie officielle CMR).
sum(adm1_utm$superficie_km2) |> round()
```

```{webr}
# On bâtit une table population minimale (clé = nom de région).
pop_tbl <- tibble::tibble(NAME_1 = regions, pop_2019 = pop_est)

# left_join = jointure SQL "à gauche" : on garde toutes les régions de adm1_utm
# et on attache la population correspondante via la clé commune NAME_1.
adm1_pop <- adm1_utm |>
  left_join(pop_tbl, by = "NAME_1") |>
  mutate(densite = pop_2019 / superficie_km2)   # densité = hab / km²

# st_drop_geometry : on retire la colonne géométrie pour avoir un tableau pur.
adm1_pop |>
  st_drop_geometry() |>
  select(NAME_1, pop_2019, superficie_km2, densite) |>
  arrange(desc(densite))
```

```{webr}
ggplot(adm1_pop) +
  # Carte choroplèthe : la couleur du polygone encode la VALEUR (ici densité).
  geom_sf(aes(fill = densite), color = "white", linewidth = 0.3) +
  geom_sf_label(aes(label = NAME_1),
                size = 2.2, colour = "black",
                fill = scales::alpha("white", 0.75), label.size = 0,
                fontface = "bold", label.padding = unit(0.08, "lines")) +
  # Palette continue + transformation racine carrée pour aplatir les écarts
  # extrêmes (Littoral et Centre écrasent sinon toutes les autres régions).
  scale_fill_viridis_c(option = "B",
                       name = "Densité\n(hab/km²)",
                       trans = "sqrt") +
  labs(title = "Densité de population par région — Cameroun (estim. 2019)",
       subtitle = "Choroplèthe — échelle racine carrée pour visibilité",
       caption = "Source : BUCREP (estim.), GADM v4.1 · IFORD × GDSG 2026") +
  theme_minimal()
```

## 7. Distance entre Yaoundé et Douala — démo CRS

```{webr}
library(sf)

# st_point(c(lon, lat)) crée 1 point ; st_sfc l'enveloppe dans une "simple
# feature collection" étiquetée par un CRS. 4326 = WGS 84 (degrés GPS).
yaounde <- st_sfc(st_point(c(11.5021, 3.8480)), crs = 4326)
douala  <- st_sfc(st_point(c(9.7679,  4.0511)), crs = 4326)

# st_distance détecte que le CRS est géographique et calcule la distance
# géodésique sur l'ellipsoïde (en mètres). Aucun besoin de reprojeter ici.
dist_m <- st_distance(yaounde, douala)
dist_m

# Comparaison avec un calcul "naïf" type Pythagore sur les degrés.
# C'EST FAUX : un degré de longitude ne mesure pas la même chose qu'un degré
# de latitude (et ça dépend en plus de la latitude).
cat("Distance naïve (degrés - sans dimension):",
    round(sqrt((11.5021-9.7679)^2 + (3.8480-4.0511)^2), 4), "\n")
cat("Distance correcte (km):", round(as.numeric(dist_m) / 1000, 1), "km\n")
```

::: {.iford-callout}
La distance Yaoundé–Douala sur la route est d'environ **240 km**. Le calcul « naïf » sur les degrés donne un nombre sans dimension — vérifiez par vous-même qu'il ne ressemble en rien aux 240 km attendus.
:::
